QUESTION 3 (AUGUST 2025)
a) Explain the concept of SQL injection and how it can be prevented in web applications. Provide an example of a vulnerable SQL query and its secure version. [4]
SQL Injection Concept:
SQL injection is a code injection technique where attackers insert malicious SQL statements into input fields to manipulate database queries. This can allow unauthorized access, data theft, data deletion, or complete database compromise.

Vulnerable SQL Query Example:

php
// Vulnerable code - DO NOT USE
$username = $_POST['username'];
$password = $_POST['password'];
$query = "SELECT * FROM users WHERE username = '$username' AND password = '$password'";
If an attacker enters ' OR '1'='1 as the username, the query becomes:

sql
SELECT * FROM users WHERE username = '' OR '1'='1' AND password = ''
This would return all users because '1'='1' is always true.

Secure Version (Using Prepared Statements):

php
// Secure code using prepared statements
$stmt = $conn->prepare("SELECT * FROM users WHERE username = ? AND password = ?");
$stmt->bind_param("ss", $username, $password);
$stmt->execute();
$result = $stmt->get_result();
Prevention Methods:

Use prepared statements with parameterized queries

Input validation and sanitization

Use stored procedures

Escape special characters

Apply principle of least privilege to database accounts

b) Design a SQL query to find the top 3 highest-paid employees from a table named "Employees" with columns EmployeeID and Salary. [6]
sql
-- Using ORDER BY with LIMIT (MySQL/PostgreSQL)
SELECT EmployeeID, Salary 
FROM Employees 
ORDER BY Salary DESC 
LIMIT 3;

-- Using ROW_NUMBER() window function (SQL Server/Oracle/PostgreSQL)
SELECT EmployeeID, Salary
FROM (
    SELECT EmployeeID, Salary,
           ROW_NUMBER() OVER (ORDER BY Salary DESC) AS rank
    FROM Employees
) AS ranked
WHERE rank <= 3;

-- Using subquery with TOP (SQL Server)
SELECT TOP 3 EmployeeID, Salary
FROM Employees
ORDER BY Salary DESC;
Explanation:

The query sorts employees by salary in descending order

LIMIT/Top restricts results to the first 3 rows

For ties, you might want to use DENSE_RANK() if you want all employees with the same salary included

c) Discuss the role of indexes in SQL databases for optimizing query performance. Provide an example of creating an index on a table column. [10]
Role of Indexes in SQL Databases:

Speed up Data Retrieval: Indexes work like book indexes - they create a data structure (usually B-trees) that allows the database to find rows quickly without scanning the entire table.

Improve Query Performance: Without indexes, the database performs full table scans, which is slow for large tables. Indexes reduce the number of disk I/O operations.

Enforce Uniqueness: Unique indexes ensure that no duplicate values exist in a column.

Support Sorting and Grouping: Indexes can speed up ORDER BY and GROUP BY operations.

Facilitate Joins: Indexes on foreign key columns speed up JOIN operations between tables.

Types of Indexes:

Single-Column Index: Created on one column

Composite Index: Created on multiple columns

Unique Index: Ensures uniqueness

Clustered Index: Determines physical order of data (only one per table)

Non-Clustered Index: Logical order separate from physical data

When to Use Indexes:

Columns frequently used in WHERE clauses

Columns used in JOIN conditions

Columns used in ORDER BY or GROUP BY

Columns with high selectivity (many unique values)

When to Avoid Indexes:

Small tables

Columns rarely used in queries

Columns with frequent INSERT, UPDATE, DELETE operations (indexes slow these down)

Columns with low selectivity (few unique values)

Example of Creating an Index:

sql
-- Creating a single-column index
CREATE INDEX idx_employee_lastname ON Employees(LastName);

-- Creating a composite index
CREATE INDEX idx_employee_name ON Employees(FirstName, LastName);

-- Creating a unique index
CREATE UNIQUE INDEX idx_employee_email ON Employees(Email);

-- Creating an index on a table with specific column order
CREATE INDEX idx_salary_desc ON Employees(Salary DESC);

-- Dropping an index when no longer needed
DROP INDEX idx_employee_lastname ON Employees;
Performance Impact Example:

sql
-- Without index - Full table scan
SELECT * FROM Employees WHERE LastName = 'Smith';
-- This scans all rows (e.g., 1,000,000 rows)

-- With index on LastName
CREATE INDEX idx_lastname ON Employees(LastName);
SELECT * FROM Employees WHERE LastName = 'Smith';
-- This uses the index to find matching rows quickly

a) Write a PHP script that takes a string as input and outputs the number of vowels in the string. For example, if the input is "Hello World", the output should be "Number of vowels: 3". [10]
php
<?php
// Method 1: Simple function using strpos
function countVowels($str) {
    $vowels = ['a', 'e', 'i', 'o', 'u', 'A', 'E', 'I', 'O', 'U'];
    $count = 0;
    
    for ($i = 0; $i < strlen($str); $i++) {
        if (in_array($str[$i], $vowels)) {
            $count++;
        }
    }
    return $count;
}

// Method 2: Using strpbrk (faster)
function countVowelsAlt($str) {
    $vowels = 'aeiouAEIOU';
    $count = 0;
    $position = 0;
    
    while (($position = strpbrk($str, $vowels)) !== false) {
        $count++;
        $str = substr($position, 1);
    }
    return $count;
}

// Method 3: Using preg_match_all (most efficient)
function countVowelsRegex($str) {
    preg_match_all('/[aeiou]/i', $str, $matches);
    return count($matches[0]);
}

// Example usage
$input = "Hello World";
$vowelCount = countVowels($input);
echo "Number of vowels: " . $vowelCount;
// Output: Number of vowels: 3

// Test with different inputs
$test1 = "PHP Programming";
$test2 = "AEIOU";
$test3 = "Why";

echo "\n" . "Vowels in '$test1': " . countVowelsRegex($test1); // 3
echo "\n" . "Vowels in '$test2': " . countVowelsRegex($test2); // 5
echo "\n" . "Vowels in '$test3': " . countVowelsRegex($test3); // 0
?>
Explanation:

Method 1 loops through each character and checks if it's a vowel

Method 2 uses strpbrk which finds any of the specified characters

Method 3 uses regular expressions for the most efficient solution

The 'i' flag in regex makes it case-insensitive

b) Write a PHP script that takes a string as input and outputs the reverse of the string. For example, if the input is "Hello World", the output should be "dlorW olleH" (or "diroW olleH" as per your question). [10]
php
<?php
// Method 1: Using strrev() function (simplest)
function reverseString($str) {
    return strrev($str);
}

// Method 2: Manual reversal using loop
function reverseStringManual($str) {
    $length = strlen($str);
    $reversed = '';
    
    for ($i = $length - 1; $i >= 0; $i--) {
        $reversed .= $str[$i];
    }
    return $reversed;
}

// Method 3: Using array functions
function reverseStringArray($str) {
    $chars = str_split($str);
    $reversedChars = array_reverse($chars);
    return implode('', $reversedChars);
}

// Method 4: Recursive reversal
function reverseStringRecursive($str) {
    if (strlen($str) <= 1) {
        return $str;
    }
    return reverseStringRecursive(substr($str, 1)) . $str[0];
}

// Example usage
$input = "Hello World";
$reversed = reverseString($input);
echo "Original: " . $input;
echo "\nReversed: " . $reversed;
// Output: diroW olleH

// Test with different inputs
$test1 = "PHP";
$test2 = "123456";
$test3 = "Racecar";

echo "\n" . "'$test1' reversed: " . reverseStringManual($test1); // PHP
echo "\n" . "'$test2' reversed: " . reverseStringManual($test2); // 654321
echo "\n" . "'$test3' reversed: " . reverseStringManual($test3); // racecaR
?>

The Modern Web Development Trinity: HTML, CSS, and JavaScript
Introduction
The modern web is built upon three fundamental technologies: HTML (HyperText Markup Language), CSS (Cascading Style Sheets), and JavaScript. Each serves a distinct but complementary purpose in creating the rich, interactive, and responsive experiences that users expect today. This essay analyzes how these technologies work together and explores best practices for building professional websites.

HTML: The Structural Foundation
Role and Purpose
HTML serves as the skeleton of every webpage. It provides semantic structure and defines content hierarchy. Modern HTML5 introduced new semantic elements that give meaning to different parts of a webpage.

Semantic Markup
Semantic HTML uses tags that describe the content they contain:

html
<header> - Contains introductory content or navigation
<nav> - Defines navigation links
<main> - Specifies the main content area
<article> - Represents a self-contained piece of content
<section> - Defines thematic grouping of content
<aside> - Contains content tangentially related to the main content
<footer> - Contains footer information
Benefits of Semantic Markup:

SEO Benefits: Search engines understand content better and rank pages higher

Accessibility: Screen readers can navigate content more effectively

Maintainability: Code is easier to read and maintain

Future-Proofing: Semantic tags are designed to be extensible

Accessibility Best Practices:

html
<!-- Use alt text for images -->
<img src="logo.png" alt="Company Logo">

<!-- Use labels for form inputs -->
<label for="name">Name:</label>
<input type="text" id="name" name="name">

<!-- Use ARIA roles when needed -->
<div role="button" tabindex="0">Click Me</div>

<!-- Use proper heading hierarchy -->
<h1>Main Title</h1>
<h2>Subsection</h2>
<h3>Sub-subsection</h3>
CSS: The Presentation Layer
Role and Purpose
CSS controls how HTML elements are displayed. It enables separation of content from presentation, allowing for consistent styling across entire websites.

Styling and Layout
CSS provides powerful layout capabilities:

1. Flexbox:

css
.container {
    display: flex;
    justify-content: center;
    align-items: center;
    gap: 20px;
}
2. CSS Grid:

css
.grid-container {
    display: grid;
    grid-template-columns: repeat(3, 1fr);
    gap: 20px;
}
3. Responsive Design:

css
/* Mobile First Approach */
.element {
    font-size: 16px;
}

/* Tablet */
@media (min-width: 768px) {
    .element {
        font-size: 18px;
    }
}

/* Desktop */
@media (min-width: 1024px) {
    .element {
        font-size: 20px;
    }
}
CSS Best Practices:

Use CSS variables for consistent theming

Follow naming conventions (BEM methodology)

Optimize for performance

Use responsive design patterns

JavaScript: The Interactivity Engine
Role and Purpose
JavaScript brings web pages to life by adding interactivity, dynamic content updates, and complex functionality.

Enhancing User Interactivity:

javascript
// Form validation
document.getElementById('myForm').addEventListener('submit', function(e) {
    e.preventDefault();
    // Validate input
    if (document.getElementById('email').value.includes('@')) {
        // Submit form
    }

// Dynamic content loading (AJAX)
fetch('/api/data')
    .then(response => response.json())
    .then(data => {
        // Update DOM with new data
        document.getElementById('content').innerHTML = data.html;
    });

// DOM manipulation
const button = document.createElement('button');
button.textContent = 'Click Me';
button.addEventListener('click', () => {
    alert('Button clicked!');
});
document.body.appendChild(button);
Modern JavaScript Features:

ES6+ syntax (arrow functions, template literals, destructuring)

Async/Await for handling asynchronous operations

Modules for organizing code

Frameworks (React, Vue, Angular) for complex applications

Examples of Complex Web Applications
1. E-commerce Platform

HTML: Product listings, shopping cart, checkout forms

CSS: Responsive grid layouts, product cards, animations

JavaScript: Real-time cart updates, search filtering, payment processing

2. Social Media Dashboard

HTML: News feed, user profiles, messaging interface

Best Practices Summary
HTML Best Practices:

Use semantic elements

Ensure accessibility (WCAG guidelines)

Validate HTML

Use modern layout methods (Flexbox/Grid)

Implement responsive design

Cross-Technology Integration:

Progressive Enhancement: Build basic functionality that works everywhere, then enhance with advanced features

Conclusion
HTML, CSS, and JavaScript form a powerful ecosystem for building modern web applications. HTML provides structure, CSS adds style, and JavaScript brings interactivity. When combined with best practices like semantic markup, accessibility considerations, and responsive design, these technologies enable developers to create sophisticated, accessible, and engaging web experiences that work across all devices.
